analyze package¶

The analyze package handles the harmonic analysis.

  • chords.py contains the principle analysis runner (dynamic programming).
  • preference_rules.py contains the preference rules.

analyze.chords module¶

class analyze.chords.ChordSpan(initial_ts, prev_cs)[source]¶

Bases: object

A ChordSpan is a series of TimeInstances that all have the same root. Each ChordSpan also maintains a pointer (prev_cs) to the previous ChordSpan computed in the song.

pr_score(m_root)[source]¶

Calculate the preference rule score, when using m_root as a root for this ChordSpan. Note this method is the core of the Preference Rule approach to Harmonic Analysis

Such an approach is heavily inspired by the work of Daniel Sleater and David Temperley at CMU in their Melisma Music Analyzer: http://www.link.cs.cmu.edu/melisma/

Parameters:m_root (Music21.note.Note) – a note representing the proposed root of this chord
Returns:the score obtained using this note as a root
